Step 1: Create a new Mule Project

We are going to use our API definition to create an implementation of our Customer API. To do this we need to import the RAML Specification from the Design Center

  1. Start Anypoint Studio from the desktop icon and select the same workspace you’ve been working (example: C:\workspaces\myworkspace).
  2. From Anypoint Studio’s menu, select File > New > Mule Project to create a New Mule project. A Window will pop-up to define the details of this new application.
  1. Give the project the name api-customer
  2. Select the Mule Server 4.x.x EE.
  3. Under API Implementation settings, check Specify API definition file location or URL and choose Design Center…​ by clicking in the (…) button on the right of the Location textbox.
  4. Once you click there, it should take you to the Browse Design Center for APIs window. If it asks you to login, use your Anypoint Platform account.

  1. You will see a list with all the APIs definitions that were created. Select Customer API and press OK
  2. If you don’t see the Customer API, press Load more projects.

  1. Once you see the Customer API version referenced in the New Mule Project dialog, Press Finish. This will create a skeleton project that implements your API.

  1. APIkit creates your new Anypoint Studio project with a generated implementation based on your API specification. The generated configuration implements the inbound HTTP listener, as well as, all the resources, request/response data type metadata, among other things.

Step 2: Run the Mule Project

Let’s explore the application that was just created.

  1. To test the API, let’s run it within Studio. Right click on the application in the Package Explorer view.
  2. Select Run As​ > Mule Application​. (Feel free to skip this if you’ve done this in the previous lab)

  1. Every time we start the application, we need to make sure that it is started successfully. Check in the Console tab that the project appears as "DEPLOYED".

  1. Once the application is deployed, Anypoint Studio will open a tab with the APIkit Consoles where we can test the application (to access the console, just click the button)

  1. Click the POST tab and test the customer creation by first clicking the TRY IT button

  1. Select the Body tab, you’ll see the request body, then click the SEND Button.
  2. You should receive the sample response defined in the API specification.

  1. Go to the console tab and press the red button to stop the Mule runtime server.

Step 3: Add Salesforce Connector

With the api view selected, click the Add Module  sign in the Mule Palette view in order to add the Salesforce Connector and Validation modules.

  1. Drag and Drop Salesforce Connector and click Finish

  1. At the end the palette should look similar to the following image.

Step 4: Replace the static data with a Connector

Now, you are ready to implement the GET resource operation using the Salesforce connector.

  1. Find the get:\customer:api-config resource flow (at the bottom of the api view)
  2. Delete the generated Transform Message step that returned the sample response from the API specification

  1. From the Mule Palette, select Salesforce​, scroll to Create Single and drag and drop the component to the flow.

Step 5: Connect to Salesforce

Now that you have your properties defined, let’s configure a new connector configuration. Go back to the Message Flow tab where you previously dropped the Salesforce connector and select it.

  1. Click in the Salesforce connector

  1. Look at the Query properties.
  2. Click on  button on the Connector Configuration.

  1. For the connector configuration we are going to put the following properties:
  1. username
  2. password
  3. securityToken

  1. Press [Test Connection…​] to check if everything is in place.

Step 6: Query data from the Account object

It's time to query data and find the customer accounts.

  1. Click in the Salesforce connector

  1. Write a query for Salesforce using SOQL syntax. For example:

SELECT Id, Name, Phone

FROM Account

WHERE BillingCountry = 'BR'

  1. Notice you must specify the fields because there  is no "*" to select all the fields.

Example of valid fields in Account object:

 Name, BillingStreet, BillingCity, BillingState, BillingPostalCode, BillingCountry, BillingLatitude, BillingLongitude, BillingGeocodeAccuracy, Phone, AccountNumber

  1. Click in the Transform Message

  1. Look at the Transform Message properties

  1. Delete all the content so that we can replace it with a new mapping.

  1. Start dragging the Payload Array<Object> to the corresponding Array<Object>

  1. Then start to map the other fields:
  1. Id → id
  2. Name → name
  3. Phone → phone

  1. Add the required mappings in case you have additional fields

  1. The mapping script is automatically generated for you

Congratulations! You queried the Salesforce object and transformed into your schema.

Step 7: List the Salesforce customers using API

Let’s explore the Mule Application (Customer API) that was just created.

  1. Make sure the application started successfully. Check in the Console tab that the project appears as "DEPLOYED".

  1. Click APIKit Console → Open Console

  1. Anypoint Studio will open a tab with the APIkit Consoles where we can test the application

  1. Click the GET tab and then click the TRY IT button

  1. There is no need to submit the optional parameters.
  2. Click Send button.

  1. The request will return 200 OK, followed by a list of the accounts.

Step 8 (Optional): Create a new Account

Now, you are ready to implement the POST resource operation using the Salesforce connector.

  1. Find the post:\customer:application\json:api-config resource flow (at the bottom of the api view)

  1. Delete the generated Transform Message step that returned the sample response from the API specification

  1. From the Mule Palette, select Salesforce​, scroll to Create Single and drag and drop the component to the flow.

  1. Click on Salesforce icon and the configuration will be shown below. Change its name to Create Account

  1. Adjacent to the Type: select Account.

  1. Drag and Drop a Transform Component and place it before the Salesforce Connector.

Now that we have configured the Salesforce connector, we need to create a mapping that will convert the JSON message from the HTTP Request to the Salesforce Account Object.

This is the mapping code that the picture reflects. For our current Lab exercise, make sure that the fields Name, BillingCountry and BillingState are included. The other fields are optional.

%dw 2.0

output application/java

---

{

        Name: payload.name,

        BillingCountry: payload.billingAddress.country,

        BillingState: payload.billingAddress.state,

        BillingStreet: payload.billingAddress.address1,

        BillingCity: payload.billingAddress.city,

        BillingPostalCode: payload.billingAddress.postalCode,

        ShippingStreet: payload.shippingAddress.address1,

        ShippingCity: payload.shippingAddress.city,

        ShippingState: payload.shippingAddress.state,

        ShippingPostalCode: payload.shippingAddress.postalCode,

        ShippingCountry: payload.shippingAddress.country,

        Phone: payload.phone,

        AccountNumber: uuid()

}

  1. Drag and Drop another Transform Component and place it in the end of the flow.

8. In this Transform, create a very simple transformation

payload

9. Open API Kit → Console and select POST

10. Click on Body and then Edit the text information. Change the name, address, contact information.

Do not use your real email or phone number.

11. Click Send button and check for 201 Created result.

Step 9: Deploy the API to Cloudhub

  1. Let’s deploy it to Cloudhub, so that the application is available on the Internet. 
  1. Let’s deploy it to Cloudhub. Right click on the project name → Anypoint Platform → Deploy to Cloud.

  1. Select an Environment: use Sandbox

  1. Now we need to specify application configuration for our API.

  1. On the application name put <<MyName-customer-api>>. You should see a green check if the name is available. If not, please use another name
  2. On Worker Size select 0.1
  1. Once the configuration is done, click on the Deploy Application button.
  2. Once the deployment is complete (it may take a few minutes), you can browse the API Console at http://<<MyName-customer-api>>.<region>.cloudhub.io/console/

Step 11 (Optional): Replace Secrets

Now that you have your properties defined, let’s configure a new connector configuration. Go back to the Message Flow tab where you previously dropped the Salesforce connector and select it.

  1. Click EDIT button on the Connector Configuration.
  2. For the connector configuration we are going to put the following properties:
  1. username: ${sfdc.username}
  2. password: ${sfdc.password}
  3. securityToken: ${sfdc.securityToken}
  1. Press Test Connection…​ to check everything is in place.

  1. Press OK (2 times).
